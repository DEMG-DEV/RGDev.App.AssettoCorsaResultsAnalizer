# Data Models — Assetto Corsa JSON Structures & Internal Types

## Source JSON Formats

### Format 1: AC Client / Offline (`race_out.json`)

This is the file generated by the Assetto Corsa game client after every session.

**Detection**: JSON root object contains `"players"` key.

**Location**: `C:\Users\{username}\Documents\Assetto Corsa\out\race_out.json`

```json
{
  "track": "silverstone",
  "players": [
    {
      "name": "Driver Name",
      "car": "ferrari_458",
      "skin": "rosso_corsa"
    }
  ],
  "number_of_sessions": 1,
  "sessions": [
    {
      "name": "Quick Race",
      "type": 3,
      "lapsCount": 5,
      "duration": 30,
      "laps": [
        {
          "lap": 0,
          "car": 0,
          "sectors": [74956, 65584, 49071],
          "time": 189611
        }
      ],
      "lapstotal": [0, 5, 4, 3],
      "bestLaps": [
        {
          "car": 0,
          "time": 142029,
          "lap": 3
        }
      ],
      "raceResult": [0, 7, 1, 2]
    }
  ],
  "extras": [
    {
      "name": "tyre",
      "tyreFL": "SM",
      "tyreFR": "SM"
    }
  ]
}
```

#### Session Type Mapping (from simresults `Helper::detectSession`)
| Type Value | Session Type |
|------------|-------------|
| 1          | Practice    |
| 2          | Qualify     |
| 3          | Race        |

#### Time Values
- All times are in **milliseconds** (integer)
- Sector times are in **milliseconds** (integer)
- A time of `-1` means DNF / invalid / no time recorded

#### Player-Car Indexing
- `laps[].car` is the **index** into the `players[]` array (0-based)
- `raceResult[]` contains player indices ordered by finishing position
- `bestLaps[].car` is the player index

#### Additional Fields in Content Manager Sessions
Content Manager saves enriched versions of this same format at:
```
%LOCALAPPDATA%\AcTools Content Manager\Progress\Sessions\YYMMDD-HHMMSS.json
```
(e.g., `C:\Users\devco\AppData\Local\AcTools Content Manager\Progress\Sessions\260619-214149.json`)

Files also include `event` (integer) in each session entry, `cuts` and `tyre` per lap:

```json
{
  "lap": 0, "car": 0,
  "sectors": [55067, 29926, 20659],
  "time": -1,
  "cuts": 2,
  "tyre": "SM"
}
```

**Session type 4** = Hotlap (in addition to 1=Practice, 2=Qualify, 3=Race)

CM JSONs include two extra top-level keys with rich metadata:

##### `__raceIni` (string — INI format embedded as escaped string)
Contains the full `race.ini` configuration used for the session:
```ini
[RACE]
TRACK=rt_suzuka
CONFIG_TRACK=suzukaeast
MODEL=ks_toyota_gt86
CARS=21
AI_LEVEL=100
PENALTIES=1

[CAR_0]
SKIN=rsrnurburg
DRIVER_NAME=David Mendez
NATIONALITY=Mexico
NATION_CODE=MEX
BALLAST=0
RESTRICTOR=0

[CAR_1]
MODEL=ks_toyota_supra_mkiv_tuned
AI_LEVEL=95
AI_AGGRESSION=0
DRIVER_NAME=Henri Salo
NATIONALITY=Finland
NATION_CODE=FIN

[LIGHTING]
SUN_ANGLE=151.16
TIME_MULT=1.0

[TEMPERATURE]
AMBIENT=26
ROAD=24

[WEATHER]
NAME=6_mid_clouds

[SESSION_0]
NAME=Track Day
DURATION_MINUTES=720
TYPE=1
```

##### `__quickDrive` (string — JSON embedded as escaped string)
Contains Content Manager's Quick Drive configuration:
```json
{
  "Mode": "/Pages/Drive/QuickDrive_Trackday.xaml",
  "CarId": "ks_toyota_gt86",
  "TrackId": "rt_suzuka/suzukaeast",
  "Temperature": 26.0,
  "Time": 80811,
  "dtv": "2026-06-19T21:20:32.117427-06:00",
  "AssistsData": {
    "IdealLine": false,
    "AutoBlip": true,
    "StabilityControl": 50.0,
    "AutoShifter": true,
    "Abs": 2,
    "TractionControl": 1,
    "Damage": 20.0,
    "TyreWear": 1.0,
    "FuelConsumption": 0.0,
    "TyreBlankets": true
  }
}
```

Key fields from `__quickDrive`:
- `dtv` — ISO 8601 datetime when the session was started (USE THIS for the session date!)
- `Mode` — session mode page (Hotlap, Race, Trackday, etc.)
- `TrackId` — includes config as `track/config` format
- `Temperature`, `Time` — weather/time settings
- `AssistsData` — detailed assist configuration

#### Content Manager Auto-Discovery Paths (Desktop)

| Priority | Path | Description |
|----------|------|-------------|
| 1 | `%LOCALAPPDATA%\AcTools Content Manager\Progress\Sessions\` | CM session history (multiple timestamped files) |
| 2 | `%USERPROFILE%\Documents\Assetto Corsa\out\race_out.json` | Vanilla AC latest session (single file, overwritten) |

#### Session Type Mapping (Server)
| Type String | Session Type |
|-------------|-------------|
| "PRACTICE"  | Practice    |
| "QUALIFY"   | Qualify     |
| "RACE"      | Race        |

#### Known Issues (from simresults source)
- `CarId` values can be **reused** across different drivers (bug in AC server)
- `Result[]` ordering is **NOT reliable** — must sort by `TotalTime` or lap count
- Some fields may be missing: `BallastKG`, `Restrictor`, `Tyre`

## Internal Data Model (TypeScript)

These are the unified types our application uses, regardless of which JSON format the data came from.

```typescript
/** Identifies which JSON format was parsed */
type SourceFormat = 'ac-client' | 'ac-server-json';

/** A single lap sector time */
interface SectorTime {
  /** Sector number (1-based) */
  sectorNumber: number;
  /** Time in milliseconds */
  timeMs: number;
}

/** A single lap driven by a participant */
interface Lap {
  /** Lap number (1-based in our model) */
  lapNumber: number;
  /** Total lap time in milliseconds (-1 = invalid/DNF) */
  timeMs: number;
  /** Sector times for this lap */
  sectors: SectorTime[];
  /** Number of cuts/off-tracks during this lap */
  cuts: number;
  /** Tyre compound used (if available) */
  tyre?: string;
  /** Whether this lap is valid (no cuts, positive time) */
  isValid: boolean;
  /** Position at end of this lap (if calculable) */
  position?: number;
}

/** A driver who participated in the session */
interface Driver {
  /** Display name */
  name: string;
  /** Steam GUID (server format only) */
  guid?: string;
  /** Team name (if any) */
  team?: string;
}

/** A vehicle / car */
interface Vehicle {
  /** Internal car model ID (e.g., "ferrari_458") */
  modelId: string;
  /** Skin name */
  skin?: string;
  /** Ballast in KG (0 = none) */
  ballastKg: number;
  /** Restrictor value (0 = none) */
  restrictor: number;
  /** Local path or data URL to car preview image (auto-discovered) */
  previewImageUrl?: string;
  /** Local path or data URL to livery thumbnail (auto-discovered) */
  liveryImageUrl?: string;
}

/** Finish status of a participant */
type FinishStatus = 'finished' | 'dnf' | 'dq' | 'none';

/** A participant in a session (driver + vehicle + results) */
interface Participant {
  /** The driver(s) — usually 1, but supports driver swaps */
  drivers: Driver[];
  /** The vehicle driven */
  vehicle: Vehicle;
  /** Final classification position (1-based) */
  position: number;
  /** Grid/starting position (1-based, if available) */
  gridPosition?: number;
  /** All laps driven */
  laps: Lap[];
  /** Best lap (fastest valid lap) */
  bestLap?: Lap;
  /** Total race time in milliseconds */
  totalTimeMs: number;
  /** Total number of laps completed */
  totalLaps: number;
  /** Number of pitstops */
  pitstops: number;
  /** Finish status */
  finishStatus: FinishStatus;
  /** Gap to leader (ms) — calculated */
  gapToLeaderMs?: number;
  /** Gap to car ahead (ms) — calculated */
  gapToAheadMs?: number;
  /** Has penalty flag */
  hasPenalty: boolean;
  /** Penalty time in ms */
  penaltyTimeMs: number;
}

/** Track information */
interface Track {
  /** Track venue name (e.g., "silverstone") */
  venue: string;
  /** Track layout/config (e.g., "full course") */
  course?: string;
  /** Track length in meters (if available) */
  lengthMeters?: number;
}

/** Server information */
interface Server {
  /** Server name */
  name: string;
}

/** Session types */
type SessionType = 'practice' | 'qualify' | 'warmup' | 'race' | 'hotlap';

/** An incident / collision event */
interface Incident {
  /** Type of incident */
  type: 'collision-car' | 'collision-env';
  /** Participant involved */
  participant: string;
  /** Other participant (car collision only) */
  otherParticipant?: string;
  /** Impact speed (km/h) */
  impactSpeed?: number;
}

/** A complete session with all its data */
interface Session {
  /** Session type */
  type: SessionType;
  /** Session name (e.g., "Quick Race", "Qualifying") */
  name?: string;
  /** Maximum laps allowed */
  maxLaps?: number;
  /** Maximum duration in minutes */
  maxMinutes?: number;
  /** Actual number of laps that occurred */
  lastedLaps: number;
  /** Track information */
  track: Track;
  /** Server info (if applicable) */
  server?: Server;
  /** Game name — always "Assetto Corsa" */
  game: string;
  /** All participants sorted by finishing position */
  participants: Participant[];
  /** All incidents */
  incidents: Incident[];
  /** Extra settings (e.g., tyre compounds, assists) */
  settings: Record<string, string | number>;
  /** Source format that was parsed */
  sourceFormat: SourceFormat;
}

/** Result of parsing a file — may contain multiple sessions */
interface ParseResult {
  /** Original file name */
  fileName: string;
  /** File size in bytes */
  fileSize: number;
  /** When the file was parsed */
  parsedAt: Date;
  /** Sessions found in this file */
  sessions: Session[];
  /** Parse warnings (non-fatal issues) */
  warnings: string[];
  /** Parse errors (if any sessions failed) */
  errors: string[];
}
```

## Car Preview Image Discovery (Local PC Only)

When the app runs on the same PC where Assetto Corsa is installed, it can **auto-discover** and display the actual rendered preview images of each car+skin combo. These are high-quality 3D renders already present on disk.

### File Structure on Disk

```
{AC_ROOT}/content/cars/{car_id}/
├── {car_id}.kn5              # 3D model (NOT used)
├── data/                     # Car physics data
├── skins/
│   ├── {skin_name}/
│   │   ├── preview.jpg       # ← HIGH-RES car render (≈640x480, ~140KB)
│   │   ├── livery.png        # ← Small livery thumbnail (≈64x64, ~20KB)
│   │   ├── ui_skin.json      # Skin metadata
│   │   └── ...               # Textures (dds, etc.)
│   └── {other_skin}/
│       └── ...
└── ui/
    └── ui_car.json           # Car display name, brand, year, description, tags
```

### AC Installation Discovery Algorithm

The app must auto-detect where Assetto Corsa is installed. Discovery priority:

| Priority | Method | Path |
|----------|--------|------|
| 1 | **Steam Registry** (Windows) | `HKCU\SOFTWARE\Valve\Steam → SteamPath` then parse `libraryfolders.vdf` for AC app ID `244210` |
| 2 | **Common Steam paths** | `D:\SteamLibrary\steamapps\common\assettocorsa\`, `C:\Program Files (x86)\Steam\steamapps\common\assettocorsa\`, etc. |
| 3 | **User-configured path** | Settings → manually set AC root folder |
| 4 | **Content Manager registry** | Read CM config for AC path if CM is installed |

### Image Resolution Logic

For each participant in parsed results:

```typescript
/**
 * Resolves the preview image path for a car + skin combination.
 * Falls back gracefully if skin or car folder is not found.
 */
function resolveCarPreview(acRoot: string, carId: string, skinName?: string): CarImages {
  // 1. Try exact skin preview
  //    → {acRoot}/content/cars/{carId}/skins/{skinName}/preview.jpg
  
  // 2. Fallback: first available skin preview
  //    → {acRoot}/content/cars/{carId}/skins/*/preview.jpg (first found)
  
  // 3. Fallback: car badge/icon from ui folder
  //    → {acRoot}/content/cars/{carId}/ui/badge.png
  
  // 4. Final fallback: generic car placeholder
  //    → embedded default car silhouette SVG
}

interface CarImages {
  /** Full car render with specific skin (~640x480) */
  previewUrl: string | null;
  /** Small livery thumbnail (~64x64) */
  liveryUrl: string | null;
  /** Whether images came from local AC install */
  isLocalAsset: boolean;
}
```

### Platform Behavior

| Platform | Preview Images Available? | Method |
|----------|--------------------------|--------|
| **Desktop (Tauri)** | ✅ Yes — auto-discover AC folder | Native file system access via Tauri API |
| **Web (same PC)** | ⚠️ Limited — File System Access API (Chrome/Edge only) | User grants folder access once, app caches |
| **Web (remote/mobile)** | ❌ No — AC not installed | Show placeholder silhouettes or generic car icons |

### Extra: `ui_car.json` Metadata

Each car also has a `ui/ui_car.json` with human-readable info:
```json
{
  "name": "Toyota GT86",
  "brand": "Toyota",
  "class": "street",
  "tags": ["street", "japan", "rwd"],
  "year": 2015,
  "description": "...",
  "bhp": "200bhp",
  "torque": "205Nm",
  "weight": "1240kg"
}
```
This can be used for **car display names** (instead of raw IDs like `ks_toyota_gt86`) and grouping/filtering.

```
┌──────────────────┐     ┌───────────────────┐     ┌──────────────────┐
│  File Input       │────▶│  Format Detector   │────▶│  Parser          │

└──────────────────┘     └───────────────────┘     └──────────────────┘
                                                            │
                                                            ▼
                                                   ┌──────────────────┐
                                                   │  Unified Model   │
                                                   │  (ParseResult)   │
                                                   └──────────────────┘
                                                            │
                                          ┌─────────────────┤
                                          ▼                 ▼
                                 ┌──────────────────┐  ┌───────────────────┐

                                          │            └───────────────────┘
                                          │                 │
                                          ▼                 ▼
                                 ┌──────────────────────────────┐

```
