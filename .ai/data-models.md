# Data Models — Assetto Corsa JSON Structures & Internal Types

## Source JSON Formats

### Format 1: AC Client / Offline (`race_out.json`) + Content Manager Sessions

This is the file generated by the Assetto Corsa game client after every session.

**Detection**: JSON root object contains `"players"` key.

```json
{
  "track": "silverstone",
  "players": [
    {
      "name": "Driver Name",
      "car": "ferrari_458",
      "skin": "rosso_corsa"
    }
  ],
  "number_of_sessions": 1,
  "sessions": [
    {
      "name": "Quick Race",
      "type": 3,
      "lapsCount": 5,
      "duration": 30,
      "laps": [
        {
          "lap": 0,
          "car": 0,
          "sectors": [74956, 65584, 49071],
          "time": 189611
        }
      ],
      "lapstotal": [0, 5, 4, 3],
      "bestLaps": [
        {
          "car": 0,
          "time": 142029,
          "lap": 3
        }
      ],
      "raceResult": [0, 7, 1, 2]
    }
  ],
  "extras": [
    {
      "name": "tyre",
      "tyreFL": "SM",
      "tyreFR": "SM"
    }
  ]
}
```

#### Session Type Mapping (AC Client)
| Type Value | Session Type |
|------------|-------------|
| 1          | Practice    |
| 2          | Qualify     |
| 3          | Race        |

#### Time Values
- All times are in **milliseconds** (integer).
- Sector times are in **milliseconds** (integer).
- A time of `-1` means DNF / invalid / no time recorded.

#### Player-Car Indexing
- `laps[].car` is the **index** into the `players[]` array (0-based).
- `raceResult[]` contains player indices ordered by finishing position.
- `bestLaps[].car` is the player index.

#### Content Manager Extras
Content Manager saves enriched versions of this same format, adding `cuts` and `tyre` per lap, as well as `__raceIni` (escaped INI file string) and `__quickDrive` (escaped JSON settings string with assists, date, and weather info).

## Internal Data Model (TypeScript)

These are the unified types our application uses on the client:

```typescript
/** Identifies which JSON format was parsed */
type SourceFormat = 'ac-client' | 'ac-server-json';

/** A single lap sector time */
interface SectorTime {
  sectorNumber: number;
  timeMs: number;
}

/** A single lap driven by a participant */
interface Lap {
  lapNumber: number;
  timeMs: number;
  sectors: SectorTime[];
  cuts: number;
  tyre?: string;
  isValid: boolean;
  position?: number;
}

/** A driver who participated in the session */
interface Driver {
  name: string;
  guid?: string;
  team?: string;
}

/** A vehicle / car */
interface Vehicle {
  modelId: string;
  skin?: string;
  ballastKg: number;
  restrictor: number;
}

/** Finish status of a participant */
type FinishStatus = 'finished' | 'dnf' | 'dq' | 'none';

/** A participant in a session */
interface Participant {
  drivers: Driver[];
  vehicle: Vehicle;
  position: number;
  gridPosition?: number;
  laps: Lap[];
  bestLap?: Lap;
  totalTimeMs: number;
  totalLaps: number;
  pitstops: number;
  finishStatus: FinishStatus;
  gapToLeaderMs?: number;
  gapToAheadMs?: number;
  hasPenalty: boolean;
  penaltyTimeMs: number;
}

/** Track information */
interface Track {
  venue: string;
  course?: string;
  lengthMeters?: number;
}

/** Session types */
type SessionType = 'practice' | 'qualify' | 'warmup' | 'race' | 'hotlap';

/** A complete session with all its data */
interface Session {
  type: SessionType;
  name?: string;
  maxLaps?: number;
  maxMinutes?: number;
  lastedLaps: number;
  track: Track;
  game: string;
  participants: Participant[];
  sourceFormat: SourceFormat;
}

/** Result of parsing a file */
interface ParseResult {
  fileName: string;
  fileSize: number;
  parsedAt: Date;
  sessions: Session[];
  warnings: string[];
  errors: string[];
}
```

## Persistent Shared Cache Model (Vercel Blob)

To enable all users of the platform to access the last 20 uploaded JSON sessions, the app implements a persistent cache.

### Backend Data Structure

Sessions are saved as files inside a private Vercel Blob store under the prefix `ac-sessions/`.

```typescript
/** Structure returned by /api/sessions */
export interface CachedFile {
  fileName: string;
  content: string;
  fileSize: number;
  cachedAt: number;
}
```

### Storage Lifecycle Flow

1. **Upload**: When a user drags a valid JSON or ZIP file, the files are parsed. Successfully parsed JSON contents are sent via POST to `/api/sessions` as `{ fileName, content, fileSize }`.
2. **FIFO Cleanup**: The Vercel serverless function lists all blobs, sorts them by upload date, and keeps the count capped at `20`. When a new session pushes the total above 20, the oldest blobs are deleted automatically using the `del` API.
3. **Retrieval**: Upon reload, GET `/api/sessions` lists all blobs, fetches their raw contents via signed `downloadUrl` tokens, and returns them to the client.
4. **Local Fallback**: If the API call fails or the Blob store token is missing, the service falls back transparently to `localStorage` under the key `ac-session-cache`.
